📌 Task 2 — Aperti Web App System (Replit Version)

Hey 👋
For this task, I need you to build a web-based version of Aperti using Replit. This is not just a basic app — it should function as a structured system for managing students, attendance, exams, and reports ⚙️📊

---

🎯 Objective

Build a web app called Aperti that:

- Manages students and groups 🧑‍🎓
- Controls access for different users 🔐
- Tracks attendance using QR codes 📷
- Stores exam & quiz results 📝
- Tracks question-level mistakes 🔍
- Generates automatic performance reports 📊

---

🧩 Core Features

1️⃣ User Roles & Access Control 🔐

The system must support:

- Admin (You) 👑
- Teacher Assistant 👨‍🏫

Each user should:

- Have their own login (username + password)
- Only access their own students
- Not see other users’ data ❌

👉 This requires:

- Authentication system
- Role-based access control

---

2️⃣ Student Management System 🧑‍🎓

Assistants should be able to:

- Add students
- Remove students
- Assign students to their account
- View their students list

Each student should have:

- Name
- Unique ID
- Group / Class
- Parent contact (optional)

---

3️⃣ Attendance System (QR Code) 📷

Build a QR-based attendance system:

- Each student has a unique QR code
- Assistant scans QR → attendance is recorded automatically
- Attendance stored with:
  - Date
  - Student
  - Status (Present / Absent)

👉 Keep it simple but reliable (no fake scans ❌)

---

4️⃣ Exams & Marks System 📝

Assistants should:

- Create exams / quizzes
- Add total marks
- Enter student scores

System should:

- Calculate percentage automatically
- Store all exam history

---

5️⃣ Question-Level Mistake Tracking 🔍

This is VERY important ⚠️

When entering marks, assistant should:

- Select which questions the student got wrong
- Assign topic for each question

System should:

- Track mistakes per question
- Track mistakes per topic
- Store everything linked to student + exam

---

6️⃣ Reports Generation 📊

System must generate automatic reports:

For each student:

- Overall performance
- Exam history
- Strengths & weaknesses (topics)
- Most frequent mistakes
- Attendance summary

Also include:

- Hardest questions (lowest success rate) 🔥
- Weakest topics across students

👉 Reports should be:

- Clean
- Readable
- Ready to export or screenshot

---

⚙️ Technical Requirements

- Build using Replit (backend + frontend) 💻
- Use a database (NOT static data ❌)
- Clean structure (routes, logic, database separated)
- Scalable design (can handle many students)
- No messy or duplicated logic 🚫

Suggested stack:

- Backend: Node.js / Python (Flask)
- Database: SQLite or PostgreSQL
- Frontend: Simple HTML/CSS/JS (clean UI)

---

💡 Important Notes

- Think like building a real product, not a school project 🧠
- Keep everything organized and modular
- UX should be simple for assistants to use
- Data accuracy is critical ⚠️

---

🚀 Goal

Build a functional Aperti web app that:

- Replaces manual sheets
- Automates tracking & reports
- Gives clear insights into student performance

This should feel like a real system ready to be used in a class environment 💼🔥

If anything is unclear, ask before starting 👍
Focus on structure, not just features 💯
