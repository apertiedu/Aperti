## 1. STUDENT HOME (StudyStream)

**Route:** `/dashboard` for student role  
**Real‑time dynamic content:**

- **Welcome Section:** “Good [morning/afternoon], [First Name]!” with current date.
- **Today’s Goal Progress:** circular progress bar showing completion percentage of daily goals (from Focus Coach).
- **Current Streak:** 🔥 icon with number of consecutive days active.
- **XP Earned Today:** from Ascend profile.
- **Exam Countdown:** if the student has an upcoming exam (from teacher’s exam schedule or student’s selected session), show a countdown timer.
- **Daily Mission Card:** one or two prioritized tasks: “Revise [weak topic]”, “Complete [assignment]”, “Review 20 flashcards”. Generated using logic: top weak topic from Echo memory + nearest overdue assignment.
- **AI Insight Card:** “You lose most marks in explanation questions.” or “Vectors mastery improved 18%.” (from Echo/ErrorTrace analysis).
- **Academic Snapshot:** four small cards showing Current Grade (average), Target Grade (set in settings), Predicted Grade (from InsightStream), Readiness Score (percentage based on attendance + completion + recent quiz scores), Attendance percentage, Homework completion rate.

**Backend:** aggregate data from Echo, Ascend, assignments, attendance, exams. If some data missing, show a friendly nudge to complete profile/onboarding.

## 3. THE MENTOR — Personalized AI Tutor

**Goal:** A conversational AI that remembers the student’s entire learning history, adapts its teaching style, and continuously helps them improve.

**Features:**
- **Learner Memory:** reads from Echo (weak topics, strong topics, learning pace, preferred explanation style, mistake history). Updates Echo after each session.
- **Teaching Modes:** the student can select or the AI auto‑selects: Explain, Simplify, Challenge, Quiz, Guide, Review.
- **Explanation Styles:** based on Echo’s `preferredStyle`, the AI adjusts: Visual (uses diagrams/analogies), Step‑by‑step, Exam‑focused, Conceptual, Practical, Analogy‑based.
- **Subject Modes:** aware of the student’s enrolled subjects (physics, math, chemistry, biology, etc.) and the current topic from the active course or recent assignment.
- **Context Awareness:** knows the current syllabus, the last assignment, recent mistakes, and teacher objectives.
- **Error Memory:** if the student repeatedly makes the same mistake (e.g., momentum calculations), the Mentor proactively revisits units, formula rearrangement, substitution errors, and exam wording.
- **Streaming chat interface** using Server‑Sent Events (SSE) or WebSocket. The frontend shows typing indicator and animated avatar.

**Backend:** use OpenAI with a detailed system prompt that includes the student’s memory and context. If no API key, use a rule‑based tutor that can answer common questions from a pre‑loaded knowledge base or by searching QueryVault for similar questions.

**API:** `POST /api/mentor/chat` (streaming), `GET /api/mentor/history`.

## 4. REVISIT — Intelligent Revision Engine

**Purpose:** Build a daily, weekly, and exam‑specific revision schedule.

**Inputs:**
- Assignment scores
- Exam scores
- Attendance records
- Flashcard retention data (from CardStack)
- Teacher recommendations
- Upcoming exam dates (from ExamScope or manual entry)

**Outputs:**
- **Daily Revision Schedule:** shown on Student Home, listing specific tasks (e.g., “Review 10 momentum flashcards”, “Complete 5 circuit questions”, “Watch force diagram video”).
- **Weekly Plan:** a calendar view with allocated slots for each subject/topic.
- **Exam Sprint Plan:** intensive revision leading up to an exam, based on predicted topics (from ExamScope) and weakness.

**Prioritization algorithm:** each topic gets a score = (exam weight * weakness level) / (time since last review). Higher score = higher priority.

**API:** `GET /api/revisit/plan?type=daily|weekly|sprint` returns structured plan.

## 9. SIMVERSE — Unified Simulation World

All four labs must be fully interactive and embedded in a unified hub.

### GEOMETRIX
- 3D shape viewer (Three.js) with all shapes: cube, cuboid, prism, pyramid, cylinder, cone, sphere, hemisphere, tetrahedron, etc.
- Interactive nets: fold/unfold animation, color faces, show vertices/edges.
- Measurements: surface area, volume, diagonal lengths.
- Vector and bearing visualizations (2D/3D).
- Dynamic graphing for functions.

### FORGEFIELD
- Physics simulations: mechanics (motion with graphs), forces, momentum, energy, electricity (circuit builder with real‑time current), magnetism (field lines), waves (amplitude/frequency controls), thermal physics.
- Adjustable variables with sliders, real‑time graph generation, data export.

### REACTSPHERE
- Chemistry simulations: atoms, molecules, bonding, reactions (mix chemicals, see precipitate/gas/color change), titrations, electrolysis, organic molecule builder.
- Lab equipment simulation.

### BIOSPHERE
- Biology simulations: cell structure (3D interactive), DNA replication, genetics (Punnett squares), ecology (predator‑prey), organ systems, enzyme activity, evolution simulation.

### PULSELAB
- Experiment evaluation mode: student performs a simulated experiment, then the system evaluates method, variables, data, graphs, conclusion, and gives feedback.

**UI:** Tabbed interface for each lab, full‑screen canvas, control panels styled with teal accents.

**API:** simulations load config from database (`simulations` table), results can be saved for teacher review.
