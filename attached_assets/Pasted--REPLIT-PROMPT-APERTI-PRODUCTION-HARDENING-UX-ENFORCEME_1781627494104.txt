You are a Senior Staff Full-Stack Engineer + QA Lead + SRE (Site Reliability Engineer) responsible for hardening the Aperti platform for real-world production use.

Your goal is NOT to add new features.

Your goal is to ensure:

system correctness

UX consistency

failure safety

observability

predictable behavior under load

⚠️ CORE RULE

Never assume anything works unless it is:

validated

logged

observable

and safely recoverable

🧠 1. GLOBAL SYSTEM VALIDATION LAYER

Implement a centralized validation layer that checks:

For every AI response:

schema correctness

confidence field exists

required metadata exists

fallback state defined

If ANY field is missing:

DO NOT crash

return safe fallback response

log structured error to system_validation_errors

🧯 2. UNIVERSAL ERROR HANDLING STANDARD

Replace all scattered error handling with a unified pattern:

Every API route must:

wrap execution in safe handler

call trackFailure() on exception

return structured fallback response

Standard fallback response:

{
status: "degraded",
message: "System temporarily operating in safe mode",
requiresReview: true,
error_code: string
}

NO unstructured errors allowed.

🔍 3. FULL OBSERVABILITY REQUIREMENT

Ensure system tracks:

Every request:

latency_ms

success/failure

model used

confidence (if AI involved)

Every AI call:

prompt_tokens

completion_tokens

response_time

confidence_score

quality_score (if available)

Store in:

system_metrics_log

Implement a UX rule validator middleware.

This ensures UI consistency across all flows:

Rules to enforce:

loading states MUST exist for async actions

every failure MUST show user-friendly message

no blank screens allowed

every AI output MUST have confidence badge if applicable

fallback states MUST be visually distinct

If a UI route violates rules:

log ux_rule_violation

Backend checks:

routes reachable
